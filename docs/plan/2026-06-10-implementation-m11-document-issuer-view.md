# M11 — 공문 발급자 열람/인쇄 + 직인 저장 확정 흐름

> 작성 2026-06-10(수). 상태: **구현 완료 — E2E 통과**. M10 사용자 피드백 반영.

## 검증 결과
- 백엔드 테스트 105개 PASS(신규 6: mine 401/본인만+sealed 플래그, issuer-view 본인/타인 404/401, 직인 교체).
- 브라우저 E2E: 직인 선택 → "미리보기 상태" 안내 + [직인 다시 선택] 유지 + [취소]/[저장] →
  저장 → "발급자가 확인할 수 있습니다" → 공문 관리 "발급한 공문" 목록(날인 완료 뱃지) →
  [보기·인쇄] → 발급자 뷰어(비밀번호 없이 직인 공문 + 인쇄 버튼) 전부 확인.

## 요구사항 (사용자)
1. **발급자(직인 요청자)도 직인 찍힌 공문을 보고 인쇄**할 수 있어야 한다(현재는 공개 URL 쪽만 인쇄 가능).
2. 직인 날인자는 **잘못 업로드할 수 있으므로**:
   - [직인 삽입] 버튼은 날인 후에도 **사라지지 않고** 다시 선택 가능해야 한다.
   - 업로드 즉시 저장하지 않고 **화면 미리보기만** → 공문 아래 **[저장]** 버튼을 눌러야
     서버에 확정 저장되고, 그때부터 발급자가 볼 수 있다.

## 설계

### 백엔드
- `GET /api/documents/mine` (인증): 내가 발급한 공문 목록
  `[{token, templateId, createdAt, sealed}]` — sealed 는 `sealContentType != null` 로 판정
  (LAZY BLOB 미로딩). 정렬: 최신순.
- `GET /api/documents/{token}/issuer-view` (인증): **본인 발급 문서만** 비밀번호 없이 열람
  (createdBy == 현재 사용자, 아니면 404 — 존재 비노출). 응답은 기존 DocumentViewResponse.
  잠금(failedAttempts) 여부와 무관 — 발급자 본인은 항상 열람 가능.
- 직인 교체: 기존 attachSeal 이 덮어쓰기 — 그대로(재저장 = 교체). 교체 테스트 추가.
- SecurityConfig 무변경: mine/issuer-view 는 permitAll 패턴(`*/view`,`*/seal`)에 안 걸려
  `/api/**` authenticated 적용(패턴 확인 완료).

### 프론트
- **공개 뷰어(/d/:token)** 변경:
  - 파일 선택 → **로컬 미리보기**(FileReader→base64)만 (인) 위에 합성, 서버 전송 X.
  - [직인 삽입] 버튼 상시 유지(선택 후엔 "직인 다시 선택").
  - 미저장 직인이 있으면 공문 아래 **[저장]** 버튼 표시 → attachSeal 호출(확정) →
    "직인이 저장되었습니다" 안내. 저장 후에도 다시 선택→저장(교체) 가능.
- **공문 관리(/documents)**: 갤러리 아래 "발급한 공문" 섹션 — 양식명/발급일/직인 상태 뱃지
  (날인 완료·대기)/[보기]. 비어 있으면 안내 문구.
- **발급자 뷰어(/documents/issued/:token)**: RequireAuth 안·AppLayout 밖(인쇄 시 메뉴 없는
  전체 화면 — 공개 뷰어와 동일 스타일). issuer-view 로 렌더 + [인쇄] [목록으로].

## 테스트 (OfficialDocumentControllerTest 추가)
1. mine: 미인증 401 / 본인 것만 목록(타인 발급 제외) + sealed 플래그
2. issuer-view: 미인증 401 / 본인 200(fields) / 타인 404
3. 직인 교체: 두 번 업로드 시 두 번째 이미지로 교체

`./gradlew test` + 프론트 빌드 + 브라우저 E2E(날인자: 미리보기→다시 선택→저장 / 발급자: 목록→열람→인쇄 버튼).
