# M13 — 공문 입력값 이력 자동완성

> 작성 2026-06-11(목). 상태: **진행 중**. 사용자 요청: 입력 필드 값을 기억하는 테이블 + 자동완성.

## 요구사항
- 공문 작성 시 입력한 값(담당자/수신/TEL/문서번호 등)을 **테이블에 기억**하고,
  다음 작성 때 **자동완성**으로 제안.

## 설계

### 백엔드 — `document_field_history` 테이블
| 컬럼 | 설명 |
|---|---|
| templateId / fieldKey / value | 유니크 (양식·필드별 값 1행 — 중복 누적 방지) |
| lastUsedAt | 재사용 시 갱신 → 최근 사용순 정렬 |

- **기록**: 발급(issue) 시 fields 순회 upsert. **개행 포함 또는 100자 초과 또는 빈 값 제외**
  (본문 같은 multiline 은 자동완성 대상이 아님 — 백엔드 휴리스틱으로 필터).
- **조회**: `GET /api/documents/field-history?templateId=X` → `{fieldKey: [value...]}`
  (필드별 최근 사용순 상위 20, 전체 최근 300행 내). 인증 필수(/api/** 기본).
- **전역 공유**: 모든 사용자의 입력을 함께 누적(사내툴 — 연락처와 동일 정책).
  짧은 단일라인 필드만 저장되므로 본문 내용 노출 없음.

### 프론트
- 작성 화면 진입 시 이력 1회 로드 → 단일라인 필드를 기존 `AutocompleteInput` 으로 교체
  (`departmentSuggestions` 포함-검색 필터 재사용). multiline(본문)은 textarea 유지.

## 테스트
1. 발급 시 이력 저장 / multiline·100자 초과 값 제외
2. 같은 값 재발급 → 중복 행 없음 + lastUsedAt 갱신(정렬 상위)
3. field-history: 미인증 401 / 템플릿별 분리 / 알 수 없는 템플릿 400

## 배포
- 새 테이블 1개 → 운영 최초 1회 DB_DDL_AUTO=update(기존 절차).
